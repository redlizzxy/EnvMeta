# EnvMeta 用户体验问卷设计（腾讯问卷版 v1）

> **目标受众**：10 名非生信研究生（环境微生物相关方向）
> **形式**：腾讯问卷（https://wj.qq.com），电子问卷，**无需线下介入**
> **总耗时**：30-40 分钟（含操作 + 填问卷 + 截图上传）
> **部署时间**：2026-04-19（周日晚）
> **回收周期**：1 周（2026-04-26 截止）

## 设计原则

1. **量化与质化并重**：Likert 1-5 评分（易统计）+ 开放式反馈（捕捉意外洞察）
2. **截图上传强制对齐**：避免"自评完成"但实际跑偏；问卷后能看图核对
3. **三大维度专项**：UI 交互 / 功能齐全 / 新手引导，各独立一个段落
4. **NPS 作北极星指标**：最终"你会不会推荐给同学"直接换算采纳意愿
5. **兼容 SUS 国际标准**：附 SUS 10 题可与竞品对照
6. **问卷 ≤ 70 题**：太长流失率大；低必填率（仅 SUS + NPS 必填）

> **SUS 计算**（问卷后台统计时）：
> - 奇数题得分 = 原始分 - 1
> - 偶数题得分 = 5 - 原始分
> - 总分 = (全部 10 题得分之和) × 2.5
> - 满分 100，> 68 = 及格，> 80 = 优秀

## 回收后的 v0.9 改进优先级框架

按反馈频次 + 严重度 4 象限分类：

```
         频次高          频次低
       ┌─────────────┬─────────────┐
严重度 │ 🔴 立即修复  │ 🟡 v0.9     │
  高  │ (P0)        │ (P1)        │
       ├─────────────┼─────────────┤
严重度 │ 🟢 v1.0      │ 🔵 backlog  │
  低  │ (P2)        │ (P3)        │
       └─────────────┴─────────────┘
```

- P0 阻塞类：错误提示看不懂 / 功能明显缺失 / 卡死不 recover
- P1 UX 痛点：术语晦涩 / 流程冗余 / 参数太多
- P2 功能完善：补充分析图 / 补充输出格式
- P3 长期路线：大规模数据性能 / 插件框架 / i18n
