## 红色信号（n=2 但两人独立同向，可信度高）

需 v2 问卷 + n ≥ 8 复核，但已可作为 app 改进的优先候选：

1. **Q15「上下文推荐 ???」含义两人都给 1 分**（最低）
   - 假设：UI 里可能有真的字面 `???` placeholder 没替换；或图标符号让用户误以为是 bug
   - 后续：grep app 代码看是否有 `???` 字面量

2. **Q44 数据准备文档完整度两人都给 1 分**
   - 用户原话（陈源 Q86）："在 MD 文件中寻找答案的任务，对于完全的外行人来说有点太深奥了"
   - 假设：`data_preparation_zh.md` 入口太隐藏 / 内容太硬核 / 没有图示导航
   - 后续：考虑改为带截图的 step-by-step "我的数据从哪里来" 引导页

3. **Q57 参数 help tooltip 有用度两人都给 1 分**
   - 假设：问号气泡 hover 不显示 / 显示了但内容是占位文本 / Mac vs Win 行为差异
   - 后续：抽查几个 widget 的 help 文本和实际 hover 行为

## v1 问卷本身的问题（已确认）

- **过长**（87 题 / 实测 40-50 min）—— 用户**反馈太长难以答完**，回收率仅 20%
- **强制截图过多**（5 任务都要截图）—— 流失率高
- **知识测验题**（Q41-43 上游工具 mapping）—— 让用户感到"我不懂这些"而中途放弃
- **多份多选 + 长 Likert 矩阵**（Q31, Q63 等）—— 视觉疲劳
