# EnvMeta — 环境微生物宏基因组可视化分析平台

## 项目细化方案 v1.0

---

## 一、项目定位与核心理念

### 1.1 一句话定位

**EnvMeta 是一个面向环境微生物研究的宏基因组下游可视化分析桌面工具，核心解决"从数据文件到发表级图形"的最后一公里问题，特色功能是自动生成微生物-环境互作的生物地球化学循环图。**

### 1.2 为什么要做这个工具

环境微生物研究（尤其是涉及污染修复、元素循环的宏基因组研究）有一个共性痛点：

- **上游流程已高度自动化**（EasyMetagenome、QIIME2、HUMAnN3、MetaBAT2 等），从测序数据到注释表格已有成熟 pipeline。
- **下游可视化仍然零散且门槛高**：研究者拿到丰度表、KO 注释表、MAG 质量表等十几个文件后，需要分别用 R/Python 写脚本生成堆叠图、PCoA、热图、网络图等，每张图调颜色字体就要花半天。
- **关键的机制总结图（生物地球化学循环图）完全依赖手工**：Illustrator / PPT 手绘，改一个通路就要重画，无法与上游数据联动。

EnvMeta 要解决的就是这个"数据已经有了，但出图太痛苦"的问题。

### 1.3 目标用户

- 环境微生物方向的硕博研究生（主力用户）
- 环境微生物相关课题组的PI和研究助理
- 环境生态、污染修复、生物地球化学方向的科研人员

### 1.4 与现有工具的差异化

| 维度 | 现有工具（Shiny-phyloseq / plotmicrobiome / Anvi'o 等） | EnvMeta |
|------|-----------------------------------------------------|---------|
| 使用形式 | R Shiny 网页 / 命令行 | 本地桌面应用（exe 或本地 Web） |
| 分析范围 | 通用微生物组（偏 16S/临床） | **环境微生物专用**（污染修复、元素循环） |
| 可视化定制 | 代码级调参或有限 GUI | **GUI 实时调参 + 自动生成可复现代码** |
| 机制图 | 无 | **一键生成生物地球化学循环图** |
| 数据→图全流程 | 需要多个工具衔接 | 单一平台完成 |

## 二、功能模块细化

### 模块 A：智能文件管理器

**用户故事**：用户从上游 pipeline（美吉/诺禾/自建 pipeline）拿到一堆 .txt / .tsv / .csv 文件，拖进 EnvMeta，软件自动识别每个文件是什么、能做什么分析。

**功能细节**：

1. **文件指纹识别引擎**
   - 基于表头关键词 + 数据特征的规则匹配系统
   - 识别规则示例：
     - `SampleID` + `Group` → metadata（分组信息）
     - 列名含 K\d{5} 格式（如 K00537）→ KEGG KO 注释表
     - `Completeness` + `Contamination` → CheckM MAG 质量表
     - 列名含门/纲/目/科/属/种层级 → 物种丰度表
     - `Bray-Curtis` 或方阵结构 → β多样性距离矩阵
     - `Shannon` / `Simpson` / `Chao1` → α多样性指数表
   - 每种文件类型配一个 JSON schema，定义必需列、可选列、数据类型、值域范围

2. **文件用途推断**
   - 识别后自动标注该文件可用于哪些分析模块
   - 示例：一个物种丰度表被识别后，系统提示"可用于：堆叠图、α多样性、β多样性 PCoA、LEfSe 差异分析、RDA 排序分析"
   - 反向提示：用户选择某个分析类型后，系统显示"还需要以下文件：xxx"

3. **文件格式兼容与预处理**
   - 自动检测编码（UTF-8/GBK）、分隔符（tab/comma）、行尾符
   - 常见格式问题自动修复（如 Windows 回车符、多余空行、BOM 头）
   - 预览功能：显示前 20 行 + 数据摘要（行数、列数、缺失值比例）

### 模块 B：可视化分析引擎

**核心思路**：EnvMeta 内置环境微生物研究常用的全套可视化分析，按照标准研究范式"Reads-based → MAG-based"分层组织。

#### B1：Reads-based 群落分析层（对应论文第二章/第三章）

| 分析类型 | 图表 | 必需输入 | 可选输入 |
|---------|------|---------|---------|
| 物种组成堆叠图 | 门/属/种水平堆叠柱状图 | 丰度表 + metadata | Top N 筛选阈值 |
| α多样性 | 箱线图 + 统计检验 | α指数表 + metadata | 检验方法选择 |
| β多样性 PCoA | 散点排序图 + PERMANOVA | 距离矩阵 + metadata | 置信椭圆、标签 |
| RDA/CCA 排序 | 环境因子约束排序图 | 丰度表 + 环境因子表 | Mantel 检验 |
| LEfSe 差异分析 | LDA 柱状图 + 进化分支图 | 丰度表 + metadata | LDA 阈值 |
| 功能基因热图 | 元素循环基因 Z-score 热图 | KO 注释表 | 自定义 KO 列表 |
| 基因差异分析 | log2FC 对比柱状图 | KO 注释表 + metadata | 比较组选择 |
| 理化指标图 | 环境参数柱状图 | 环境因子表 | — |

#### B2：MAG-based 基因组分析层（对应论文第四章）

| 分析类型 | 图表 | 必需输入 |
|---------|------|---------|
| MAG 质量评估 | Completeness vs Contamination 散点图 | CheckM 结果表 |
| MAG 丰度热图 | 聚类热图 + 门注释 + 组间丰度条 | 丰度表 + 分类注释 |
| 代谢通路完整度 | 气泡矩阵图（MAG × 通路） | KO 注释表 + 分类注释 |
| MAG 元素循环基因谱 | 基因有无/拷贝数热图 | KO 注释表 + 分类注释 |
| 共现网络图 | 力导向网络 + Keystone 标注 | 丰度矩阵（或预计算的相关性矩阵） |

#### B3：分析代码管理

- **预设代码库**：每种分析类型内置经过验证的 Python 脚本（基于你现有的 01-09 系列脚本改造）
- **用户自定义代码**：用户可上传自己的 .py 脚本，系统解析其输入参数并匹配文件
- **代码版本管理**：每次调参都保存为一个代码快照，用户可回溯

### 模块 C：交互式图形调参系统

**核心理念**：所有调参操作最终都映射为代码参数的修改。用户看到的是 GUI 控件，背后是一个参数字典在变化，最终输出的代码包含所有修改。

**通用调参面板**（所有图表共享）：

| 参数类别 | 控件形式 | 参数示例 |
|---------|---------|---------|
| 画布 | 数值输入框 + 预设 | 宽度、高度、DPI、边距 |
| 字体 | 下拉选择 + 大小滑块 | 字体族、标题字号、标签字号、加粗/斜体 |
| 配色 | 色板选择器 + 预设方案 | 分组颜色、热图色阶、渐变方案 |
| 图例 | 位置选择 + 开关 | 显示/隐藏、位置（9 宫格）、列数、字号 |
| 轴 | 文本输入 + 范围 | 轴标签文本、刻度范围、刻度间隔、旋转角度 |
| 输出 | 格式选择 | PDF / SVG / TIFF / PNG、DPI 设置 |

**特定图表调参**（举例）：

- 堆叠图：Top N 物种数、"其他"分类合并阈值、柱宽、间距
- 热图：聚类方法（ward/complete/average）、距离度量、行列标签筛选、色阶断点
- 网络图：布局算法（spring/kamada_kawai/circular）、节点大小映射、边过滤阈值
- PCoA：置信椭圆样式、标签显示规则、轴选择（PC1/PC2/PC3）

**工作流程设计**：

```
用户选择分析类型 → 匹配输入文件 → 运行预设代码生成初始图 →
→ GUI 调参面板出现 → 用户调整参数 → 点击"刷新预览" → 图表实时更新 →
→ 满意后点击"保存" → 系统生成包含所有修改的完整 .py 脚本 + 高质量图形文件
```

**关于"实时预览"的两种实现策略**：

- **策略 A（推荐）：延迟批量更新**。用户调完所有参数后，点击"重新生成"按钮，一次性重新运行脚本。优点是实现简单、不卡顿；缺点是不能实时看效果。
- **策略 B：真正实时预览**。每次参数变化后自动重新渲染低分辨率预览图（缩小画布 + 降低 DPI）。最终导出时再用完整参数渲染。需要做防抖（debounce）优化。

建议先实现策略 A，后续优化为策略 B。

### 模块 D：高质量导出系统

- **图形导出**：PDF（矢量，fonttype=42）、SVG、TIFF（600dpi）、PNG（300dpi）
- **代码导出**：每张图导出对应的完整 .py 脚本，包含所有调参结果，可独立运行
- **批量导出**：一键导出所有图表 + 代码 + 统计结果汇总表
- **论文就绪预设**：
  - "毕业论文版"：中文标注、300dpi PDF、宋体+Times New Roman
  - "SCI 投稿版"：英文标注、600dpi TIFF、Arial

### 模块 E：生物地球化学循环图生成器 ⭐（核心创新）

这是整个项目最有价值、也最具挑战性的功能。下面详细拆解。

#### E1：数据输入与通路推断

**输入文件**：
- KEGG KO 注释表（必需）——提供功能基因信息
- 物种丰度/分类注释（必需）——提供"谁在做"
- 环境因子表（推荐）——提供环境背景
- Keystone 物种列表（可选）——高亮关键物种

**自动推断逻辑**：

1. 程序内置一个"元素循环知识库"（你已有的 KO 字典的扩展版），结构如下：
   ```
   元素 → 代谢过程 → 子通路 → KO 列表 → 基因名 → 反应描述（底物 → 产物）
   ```
   覆盖范围建议从你论文涉及的 4 大元素出发，逐步扩展：
   - **第一梯队（你的研究）**：As（砷）、S（硫）、Fe（铁）、N（氮）
   - **第二梯队（环境研究高频）**：C（碳）、P（磷）、Mn（锰）、Se（硒）
   - **第三梯队（扩展）**：Hg（汞）、Cr（铬）、U（铀）及其他重金属

2. 扫描用户的 KO 注释表，统计每个元素循环通路的基因检出情况
3. 结合物种信息，建立"物种 → 基因 → 通路 → 元素"的四层映射
4. 输出一个"循环图数据模型"：哪些元素通路是活跃的、由哪些物种驱动

#### E2：循环图的视觉模型设计

参考你论文中的 Fig3-9（机制图）以及 Zhu et al. 2017 的砷循环图，循环图应包含以下可视化层次：

**双层结构**：
- **外层——环境层**：展示样本环境中的物质形态和转化（如 As(V) ↔ As(III)、SO₄²⁻ ↔ S²⁻、Fe(III) ↔ Fe(II)、NO₃⁻ → N₂ 等）
- **内层——微生物层**：展示关键微生物及其参与的代谢过程，用箭头连接到环境层中的物质转化

**视觉元素**：
- 环境物质：圆角矩形节点，颜色按元素分类（As=红、S=黄、Fe=橙、N=蓝）
- 微生物：椭圆节点，大小映射丰度，颜色映射门水平
- 代谢箭头：连接微生物与物质转化，标注关键基因名
- 通路完整度：箭头粗细或透明度映射通路完整度百分比
- 组间差异：可切换不同实验组，对比同一循环在不同处理下的变化

#### E3：交互式编辑功能

用户在自动生成的循环图基础上进行调整：

- **节点编辑**：拖拽位置、修改标签、更改颜色/大小、添加/删除节点
- **通路编辑**：连接/断开箭头、修改箭头标注（基因名/反应名）、调整通路方向
- **布局调整**：选择预设布局模板（双层环形、左右分区、自由布局）
- **注释添加**：添加文本框、虚线分隔区域、图例说明
- **组间切换**：切换到不同实验组的数据，或显示组间差异的overlay
- **元素筛选**：勾选要展示的元素循环（如只看 As + Fe 的耦合）

#### E4：技术实现建议

循环图的交互编辑是最大挑战。有两条可行路径：

**路径 A：Web 技术（D3.js / React Flow）嵌入桌面应用**
- 用 D3.js 或 React Flow 实现节点-边的拖拽编辑
- 通过 PyQt6 的 QWebEngineView 或 Streamlit 的 iframe 嵌入
- 优点：交互体验好，拖拽/缩放天然支持
- 缺点：需要前端开发能力

**路径 B：基于 matplotlib + 自定义交互层**
- 用 matplotlib 的 event handler 实现节点拖拽
- 参考你现有的 `09_cooccurrence_network.py` 的绘图架构
- 优点：与现有代码一致，不引入新技术栈
- 缺点：交互体验受限，复杂编辑难实现

**建议**：路径 A 做编辑，路径 B 做最终高质量导出。即用 D3.js 做编辑界面，确定布局后传参数给 matplotlib 渲染最终的出版级 PDF。

## 三、技术架构优化建议

### 3.1 重新审视"是否一定要做 exe"

基于搜索到的现有工具生态和你的实际需求，我提出一个**可能更优的方案**：

**方案：Streamlit + 本地启动（取代传统 exe）**

理由：
1. 你的目标用户是研究生/科研人员，他们的电脑上通常已有 Python 环境
2. Streamlit 的开发效率是 PyQt6 的 3-5 倍，且 UI 天然美观
3. Streamlit 支持文件上传、参数滑块、图表展示、代码高亮——完美匹配你的需求
4. 可以通过 `streamlit run app.py` 一键启动，体验接近桌面应用
5. 如果未来需要，可以用 `pyinstaller` 打包 streamlit 启动脚本，或用 `stlite` 做 WASM 版本
6. 循环图编辑器可以直接用 `streamlit-agraph` 或自定义 HTML 组件嵌入 D3.js

**对比**：

| 维度 | PyQt6 exe | Streamlit 本地 Web |
|------|-----------|-------------------|
| 开发速度 | 慢（GUI 手写） | 快（声明式 UI） |
| UI 美观度 | 中等 | 高 |
| 文件上传 | 需手写对话框 | `st.file_uploader()` 一行 |
| 图表展示 | matplotlib 嵌入 | Plotly 交互式图表 |
| 参数调节 | 手写控件 | `st.slider()` / `st.color_picker()` |
| 代码展示 | 需嵌入文本编辑器 | `st.code()` 语法高亮 |
| 打包分发 | PyInstaller → ~300MB exe | pip install + streamlit run |
| 循环图编辑 | 需嵌入 QWebView | 直接嵌入 HTML 组件 |

### 3.2 如果坚持 exe 方案

如果你确定需要打包为独立 exe（比如要分发给完全不懂编程的用户），建议：

- 使用 **Nuitka**（而非 PyInstaller），编译为 C 再打包，体积可缩小 30-50%
- 前端框架选 **PyWebView + Flask**：用 pywebview 创建一个原生窗口，内部跑 Flask 后端 + HTML/JS/CSS 前端，这样兼顾了 Web 技术的灵活性和桌面应用的体验
- R 脚本全部改写为 Python，消除对 R 环境的依赖

### 3.3 推荐的最终技术栈

```
┌─────────────────────────────────────────────┐
│               EnvMeta 技术栈                  │
├─────────────────────────────────────────────┤
│  前端 UI：Streamlit（或 pywebview + HTML）    │
│  交互图表：Plotly（替代 matplotlib 做预览）    │
│  高质量输出：matplotlib（最终渲染出版级图）    │
│  循环图编辑器：D3.js（嵌入 st.components）    │
│  后端逻辑：Python（pandas/scipy/networkx）    │
│  R 分析：rpy2 调用或全部改写为 Python          │
│  打包（可选）：Nuitka 或 pip 分发              │
└─────────────────────────────────────────────┘
```

## 六、值得考虑的额外优化

### 6.1 AI 辅助功能（可选，但有发表价值）

如果你有 API 额度，可以在以下环节嵌入 LLM 辅助：
- **智能文件识别**：对于无法通过规则匹配的非标准文件，调用 LLM 分析表头含义
- **循环图解读**：生成循环图后，自动用 LLM 写一段结果描述（如"在B组中，硫氧化细菌 Sulfuricaulis 的丰度显著升高，其 sox 通路完整度达 85%，表明..."）
- **图表标注建议**：根据统计结果自动建议在图上标注哪些显著差异

### 6.2 与上游 pipeline 的衔接

考虑直接支持以下格式的输入：
- **美吉云平台**导出的报告目录结构
- **EasyMetagenome** 的标准输出结构
- **HUMAnN3** 的 pathabundance / genefamilies 表
- **MetaPhlAn4** 的 profiled_metagenome 表
- **CheckM2** / **GTDB-Tk** 的标准输出格式

这样用户不需要做任何格式转换，直接把上游输出文件夹拖进来即可。

### 6.3 发表策略

这个工具本身可以作为一个 **Methods/Application 论文** 发表：
- 目标期刊：Bioinformatics、Briefings in Bioinformatics、iMeta、Frontiers in Microbiology (Bioinformatics 专栏)
- 核心卖点：首个面向环境微生物研究、集成生物地球化学循环图自动生成的可视化平台
- 如果用 AI 辅助功能则增加亮点

## 七、总结

你的项目本质上是在做三件事：

1. **标准化**——把环境微生物研究中零散的可视化脚本标准化为一个统一平台
2. **图形化**——把命令行操作变成 GUI 操作，降低使用门槛
3. **创新化**——生物地球化学循环图的自动生成是目前的空白，填补了这个空白

其中第 3 点是最大的创新点和差异化优势。建议开发过程中始终以此为核心，前两点是基座，第三点是皇冠上的宝石。
